Project Overview

An academic and practical project focused on identifying network vulnerabilities, auditing open ports, and diagnosing connectivity issues. Using tools like Nmap, the project mapped out the topology and security posture of targeted testing environments.

The Problem

Without proper auditing, local and remote networks can have unknowingly exposed services. Understanding exactly which ports are open and how firewalls are responding is crucial for maintaining a secure and reliable IT infrastructure.

Methodology

The approach involved methodical port scanning to identify active services (like SSH, HTTP, etc.) and analyzing the responses. Firewall rules were then reviewed to ensure that only intended traffic could pass through, providing a safer network environment and detailed diagnostic reports.

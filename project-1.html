Smart Fan IoT System (Kipas-01)

An automated smart cooling system powered by ESP32 that adjusts to room temperature in real-time, featuring MQTT communication, cloud data logging via Supabase, and a web-based manual override.

Role IoT Developer / System Integrator
Platform Hardware & Web Dashboard
Tech Stack
ESP32
DHT11 Sensor
MQTT (HiveMQ)
Supabase
JavaScript

Project Overview

This project involved designing and developing an automated cooling solution designated as "Kipas-01". It utilizes an ESP32 microcontroller paired with a DHT11 sensor to monitor environmental conditions. Hardware logic operates autonomously based on temperature, while data is transmitted in real-time via MQTT (HiveMQ). The backend features a well-structured database architecture, implementing robust one-to-many relationships in Supabase to securely log continuous sensor data. Additionally, a responsive web interface was developed using JavaScript to allow users to manually override the automated system.

The Problem

Traditional cooling fans require manual operation and lack environmental awareness, leading to inefficient energy use and inconsistent room comfort. Without a centralized data logging and remote control system, users cannot monitor temperature trends or manage the device from afar. A comprehensive IoT solution was needed to automate temperature regulation, provide reliable remote access, and systematically store historical telemetry data for future analysis.

Methodology

Development began with assembling the hardware components (ESP32 and DHT11) and programming the temperature-based conditional logic. To achieve low-latency connectivity, MQTT protocol was implemented for seamless device-to-cloud communication. On the software side, Entity-Relationship Diagrams (ERD) were mapped out to structure the Supabase backend efficiently. Finally, JavaScript was utilized to integrate the web dashboard, ensuring that telemetry logs were accurately recorded and the manual override function executed commands instantaneously.
